1. Elevate On Save: If the 'Access Denied' error is encountered while saving, offer to launch elevated to be able to save (possibly with intermediate save to %temp% and then elevating just to move that file, not for the whole editing session)
2. Templates - a powerful way of inserting almost anything:
   - Defined in the INI file similarly like filters (numbered, having name, description and category), available through the Tools menu as well.
   - A template value (or how it would be called) can contain freeform text in quotes as well as variables like %varname%, where varname is a name of a built-in (internal) variable. It's up to our imagination what the internal variables could retrieve and then insert into the template string (and thus into the document). I will elaborate on the %date% and %time% variables later. There should be also a variable, or another character or characters, that, contained in the template string, would signal the current cursor position, so that the cursor would be positioned at that place after inserting the expanded template string.
   - The template can be associated with a file extension like "md" for MarkDown, such that only templates with the given extension would be available when a document of the particular file type was loaded.
   - To make the template file types usable even more, the File | New menu item would get a submenu with the category names of templates having file types associated, e.g. Markdown. Thus, when you chose the File | New | Markdown menu item, the new Markdown document would be opened, having the associated file extension preselected for saving the file as with the default TXT. If no templates having categories and file types were available, the New menu items would have no submenu and would work as of yet. The Open and Save dialogs would then get the file type filters based on the defined categories and file extensions within templates. The Plain Text built-in template has to be available at all times to not lose the ability of saving and opening TXT files. Of course, any template could be written to be usable for the TXT file extension. On the other hand, if the template does not specify the file type, it has to be available at all times.
   - To establish a solid base to demonstrate templates, let's work out a set of Markdown templates that would insert various formatting elements into a document, making use of the cursor position denoting element so that one could write the heading text right upon inserting the heading markup, for example.
3. Configurable Date/Time Format: Let the Insert Date/Time feature be configurable via INI file parameters specifying separate strings for date, time, and template for them (like "%date%, %time%").
   - Documentation:
     - To control the %DATE%: https://learn.microsoft.com/en-us/windows/win32/api/datetimeapi/nf-datetimeapi-getdateformatex
       - Variables %LONGDATE%, %SHORTDATE%, %YEARMONTH% and %MONTHDAY% can be used in INI file template and will correspond to individual dwFlags.
       - Format String Description: https://learn.microsoft.com/en-us/windows/win32/intl/day--month--year--and-era-format-pictures
     - To control the %TIME%: https://learn.microsoft.com/en-us/windows/win32/api/datetimeapi/nf-datetimeapi-gettimeformatex
3. Support Tab Completion For Active REPL Filter: Triggered by pressing TAB while on the Prompt Line after the detected prompt; sends TAB to the REPL Filter and captures its stdout, inserting it into the current line at the cursor position
