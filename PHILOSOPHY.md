# Project Philosophy

RichEditor is intentionally focused. If you fork it and choose a different direction, please document the divergence and adjust the name to avoid confusion. This is a request, not a legal requirement.

## Core Principles

- **Win32-first:** native Win32 UI and APIs, minimal dependencies.
- **Accessibility first:** screen reader-friendly menus and UI behavior are non-negotiable.
- **Minimal design:** avoid packing every feature; prefer a clean, deliberate feature set.
- **Extensibility over bloat:** rely on external tools (filters, scripting) rather than hardcoded features.
- **Small executable size:** prefer size over micro-optimizations; keep the binary lean.
- **Thoughtful features:** add small, clever improvements or well-designed extensions that unlock user workflows.

## Forking Request

If your fork diverges from these principles:

1) Please **rename** the project.
2) Please **document** the philosophical differences in your README or docs.

Thanks for respecting the spirit of the project.
